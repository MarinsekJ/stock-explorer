The machine learning pipeline of the stock-explorer application is divided into distinct offline training and live runtime inference environments. The predictive model is trained once offline using historical data gathered from the data source API. This trained asset bridges the gap between the two phases by being saved as a serialized file (such as a .pkl file) and loaded by the application backend into memory at startup. Live inference happens dynamically at runtime through a backend prediction endpoint whenever a user triggers a request. The endpoint processes the query using the loaded model and instantly serves the resulting stock market projections to the frontend interface.
